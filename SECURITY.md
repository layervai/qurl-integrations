# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Private Vulnerability Reporting**: Use the [Security tab](https://github.com/layervai/qurl-integrations/security/advisories/new) to privately report a vulnerability.
2. **Email**: Send details to [security@layerv.ai](mailto:security@layerv.ai).

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to expect

- Acknowledgement within 48 hours
- Regular updates on the fix timeline
- Credit in the fix release (unless you prefer anonymity)

## Supported Versions

We release patches for security vulnerabilities in the latest version of each integration. Older versions are not actively supported.
